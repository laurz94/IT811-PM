Requirements

  1. Ease of Use

    1. Responsiveness: The site should be responsive to the screen size. The following sizes should be utilized:
      • Desktop -1920×1080
      • Tablet - 1366×768
      • Mobile - 640×360
    2. Browser Compatibility: The site should be compatible with the following evergreen browsers:
      • Chrome
      • Edge
      • Firefox
      • Safari
    3. Navigation: The menu should provide categories of pages. Links to individual pages should be listed in the sub-menu. The menu layout should be:
      • Apparel
        • Men
        • Women
        • Lifestyle
        • Students
      • Events
        • Calendar
        • Tickets
      • Books
        • Book Search
        • Order
      • Services
        • Printing
        • Engraving
      • Community Hub

      Additionally, the site should contain the following items in the navigation bar

      • Search
      • Cart
      • Account Login

      4. Definition of Done

      The user can easily navigate the website and find what they need quickly. Navigation is logically organized. The site is responsive to screen size on which it is displayed, and the user does not need to pan or zoom to navigate the website. The website is accessible from an evergreen browser.

  2. Performance

    1. The site should respond to a request in no longer than 2 seconds from 10 mbps ethernet connection or 4G mobile connection.

    Definition of Done

    The response time of application is maximum 2 seconds.

  3. Multiple Payment Methods

    1. The site should be able to accept the following payment methods:
      • Visa
        • Debit & Credit
      • Mastercard
        • Debit & Credit
      • American Express
        • Debit & Credit
      • Discover
        • Debit & Credit
      • Paypal
      • Cryptocurrency
      • Android Pay
      • Apple pay
      • Beak’em Bucks
    2. The site should integrate with the Stripe API to provide secure payments.

    Definition of Done

    All requested payment methods have been implemented and can be used successfully.

  4. Simulated Merchandise Preview

    1. The user should be able to upload an image.
    2. The site should be able to place that photo on specified merchandise and show a rendering to the user.
    3. The merchandise picture should have a static place for the uploaded picture.
    4. The user will not be able to move the picture on merchandise.
    5. The user should crop the picture before uploading to the site. Cropping images is out of scope.
    6. Stock images of the merchandise with a placeholder for the uploaded image will be provided by the marketing team.
    7. The user should be able to click the rendered image to enlarge the image.

    Definition of Done

    Users can upload a picture of themselves and see themselves on KU merchandise.

  5. Ticket Sales

    1. User should be logged into their KU account to view events
    2. Provide an event calendar for all KU Athletics
    3. Clicking an event on the calendar will deep link into KU Athletics website to purchase tickets.
      • Example link: https://kuathletics.evenue.net/cgi-bin/ncommerce3/SEGetEventInfo?ticketCode=GS%3AKU%3ASB22%3A01%3A&linkID=kansas&shopperContext=&pc=&caller=&appCode=&groupCode=&cgc=

    Definition of Done

    Students, faculty, and other KU staff can successfully order tickets for sports and other events

  6. Automatic Price Matching

    1. The site should automatically price match merchandise & equipment with the following websites:
      • Amazon - Books, Apparel, Equipment
      • Rally House - Apparel
      • Chegg - Books
      • Apple - Equipment
      • Best Buy - Equipment

    Definition of Done

    Prices are matched to current prices from the listed websites.

  7. Community Hub

    1. The site should offer a forum to be utilized by students and staff for any academic purpose. The user will use their existing KU login.
    2. The following are user roles for the forum:
      • Administrator - Director of KU Bookstore
        • User Management
      • Staff - KU Bookstore Staff
        • Moderation
        • User Management
      • Faculty
        • Moderation
      • Student
        • Community Hub Access

    Definition of Done

    Students can log into a community forum online and communicate with other student, staff, and faculty.

  8. Discount Hub

    1. The site should offer a list of establishments that offer discounts to Users for the following:
      • Restaurants
      • Electronics
      • Clothes
      • Events
      • Books
      • Travel
    2. Additionally, the site should contain a page to purchase clearance merchandise.

    Definition of Done

    Users will receive a list of discount retailers, and the ability to purchase clearance merchandise in the KU Bookstore.

  9. In-text Book Search

    1. The site should offer the ability to search for category based keywords in textbooks. The site should return the results by searching the text of all books for the specified key words.
    2. The user will specify the category of books to search. For example, economics majors should only search in economic books.
    3. The user should have the ability to further refine the search by
      • Date
      • Author
      • Publisher
      • Type of publication (book, magazine, journal, etc)

    Definition of Done

    Users can search for a keyword in the search bar, and then books with that keyword are displayed filtered by category and other optional filters.

  10. Outdoor Pick-up/Drop-Off

    1. The site should allow the user to schedule:
      • Book return & pickup
      • Repair drop off & Pickup
      • Merchandise pickup

    Definition of Done

    Users can schedule pick-up / drop-off on the KU bookstore website.

  11. Online Course Signup

    1. The site should offer the ability to enroll in courses.
    2. Link to the course selection page of Enroll & Pay.
    3. The user can enroll and pay for classes on the Enroll & Pay site.

    Definition of Done

    Students can join courses and access Enroll & Pay directly on the KU Bookstore website.

  12. Feedback Functionality

    1. The site should support user feedback through the following channels:
      • Chat - With KU bookstore staff.
      • Phone
      • Email - KU Bookstore staff. An email mailbox will need to be setup by IT
    2. The user should have the ability to review merchandise.
    3. The user must be logged into their KU account to create a review.

    Definition of Done

    • Users can submit feedback on the KU Bookstore website.
    • Users can chat with bookstore staff to answer questions.
    • Users can call the phone number listed on the site to speak with KU bookstore staff.
